# テストコード規約

## 必須要件

- テストは最低3本、必ず**異常系を1本以上**含める
- テスト名は日本語OK（何を検証しているか明確にする）
- `npm test` で全件パスすることを確認してからPR

## テスト構造

```typescript
describe('機能名', () => {
  // 正常系
  it('正常な入力で期待通りの結果を返す', () => { ... });

  // 異常系
  it('不正な入力でエラーを返す', () => { ... });

  // 境界値
  it('空文字列で適切に処理する', () => { ... });
});
```

## バグ修正時のテスト

バグを修正したら、必ず**再発防止テスト**を追加すること：
1. バグが再現する条件のテストを書く
2. 修正前にテストが失敗することを確認
3. 修正後にテストが通ることを確認
